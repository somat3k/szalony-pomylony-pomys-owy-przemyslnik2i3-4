# CLI Reference

The `hololang` CLI provides commands to run programs, check syntax, explore skills, and render canvases.

## Commands

### `hololang run FILE`

Execute a `.hl` program and print the output.

```bash
hololang run examples/hello_holo.hl
hololang run examples/full_system.hl
```

### `hololang check FILE`

Parse a `.hl` file for syntax errors without running it.

```bash
hololang check examples/laser_config.hl
# ✓  examples/laser_config.hl – OK
```

### `hololang repl`

Start an interactive read-eval-print loop.

```bash
hololang repl
# HoloLang REPL v0.1.0 — type :quit to exit
# hl> let x = 42
# hl> print(x)
# 42
```

### `hololang info`

Print version number and a list of all subsystems.

```bash
hololang info
# HoloLang v0.1.0
# subsystems: lang · tensor · device · mesh · vm · network · docs
```

### `hololang skills`

List all skills registered in the skill registry.

```bash
hololang skills
# laser_control       – LaserDevice control
# mirror_control      – GalvanizedMirror axis control
# tensor_pool         – Tensor / SafeTensor pool management
# mesh_canvas         – Mesh MDI canvas
# ...
```

### `hololang canvas FILE [--no-color]`

Execute a `.hl` file and render the resulting mesh canvas in the terminal.

```bash
hololang canvas examples/hello_holo.hl
hololang canvas examples/full_system.hl --no-color
```

## Full Command Reference

| Command | Flag | Description |
|---|---|---|
| `run FILE` | — | Execute `.hl` program |
| `check FILE` | — | Syntax-check only |
| `repl` | — | Interactive REPL |
| `info` | — | Version + subsystem list |
| `skills` | — | Registered skill list |
| `canvas FILE` | `--no-color` | Run + render mesh canvas |

## Exit Codes

| Code | Meaning |
|---|---|
| `0` | Success |
| `1` | Runtime / interpretation error |
| `2` | Syntax / parse error |
| `3` | File not found |
