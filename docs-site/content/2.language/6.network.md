# Network Stack

HoloLang's network layer is protocol-agnostic. Declare channels, webhooks, and APIs, then connect and emit to them with pipe expressions.

## Channel

```hololang
channel HoloGRPC(grpc) {
    host: "localhost"
    port: 50051
}

channel HoloWS(ws) {
    host: "localhost"
    port: 8080
}
```

Supported protocols: `grpc`, `ws` (WebSocket), `http`.

> **Note:** `host` and `port` inside a channel block are local assignments in the channel's scope. Default values are derived from the protocol: gRPC → port 50051, WebSocket → port 8080, HTTP → port 80. Matching the defaults in the block documents intent even if the runtime uses constructor defaults.

## Webhook

```hololang
webhook StatusHook {
    url:    "http://localhost:9000/events"
    events: "all"
}
```

> **Note:** `url` and `events` inside a webhook block are local scope assignments. The Webhook object's `url` and `events` fields should be set via `self.url = ...` if you need them reflected in the runtime object.

## REST API

```hololang
api HoloAPI {
    base_path: "/api/v1"
    port:      8000
}
```

## Connecting

```hololang
connect HoloGRPC:50051
connect HoloWS:8080
```

## Emitting Messages

```hololang
emit HoloGRPC -> "system_ready"
emit StatusHook -> "full_system_online"
```

## Piping through the Network

```hololang
invoke(HoloMesh) -> HoloGRPC -> HoloAPI
```

This invokes the mesh, sends the result over gRPC, and exposes it on the REST API.

## Listening

```hololang
listen HoloWS -> handle_incoming
```

The handler `handle_incoming` must be a previously declared `fn`.
