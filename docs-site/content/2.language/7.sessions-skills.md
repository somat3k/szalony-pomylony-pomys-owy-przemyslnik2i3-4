# Sessions & Skills

Sessions provide a named execution context for a HoloLang program. Skills are tagged capabilities that let the runtime know which subsystems a session requires.

## Session Annotation

```hololang
@session("my-project")
```

Attaches the current file execution to the named session.

## Session Block

```hololang
session MainSession {
    skill: "laser_control"
    skill: "mirror_control"
    skill: "tensor_pool"
    skill: "mesh_canvas"
    skill: "kernel_programming"
    skill: "grpc_channel"
    skill: "websocket"
}
```

> **Note:** `key: value` pairs inside a session block are evaluated as local assignments in the session's scope. Use the `hololang skills` CLI command to see all skills registered globally in the skill registry.

## Built-in Skills

| Skill key | Subsystem |
|---|---|
| `"laser_control"` | LaserDevice |
| `"mirror_control"` | GalvanizedMirror |
| `"sensor_reading"` | Sensor |
| `"tensor_pool"` | Tensor / SafeTensor pool |
| `"mesh_canvas"` | Mesh MDI canvas |
| `"kernel_programming"` | Kernel VM |
| `"grpc_channel"` | gRPC network channel |
| `"websocket"` | WebSocket channel |
| `"api_design"` | REST API |
| `"webhook_dispatch"` | Webhook dispatcher |
