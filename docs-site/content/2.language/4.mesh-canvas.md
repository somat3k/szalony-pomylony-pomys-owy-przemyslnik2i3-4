# Mesh Canvas

The `mesh` declaration creates a sparse 2-D tile canvas. Each tile can be connected to a device, tensor, or other named entity. Impulses propagate data between tiles.

## Declaring a Mesh

```hololang
mesh HoloMesh(FrameBuffer) {
    tile(0, 0) -> GreenLaser
    tile(0, 1) -> MirrorH
    tile(1, 0) -> SensorA
    tile(1, 1) -> FrameBuffer
}
```

The optional argument `(FrameBuffer)` sets the backing tensor for the canvas.

## Tile Wiring

`tile(row, col) -> target` wires a grid position to any named entity — devices, tensors, enums, pools, channels, or string labels.

## Impulse Cycles

`impulse` propagates a signal from one tile to another with an optional payload:

```hololang
impulse(tile(0, 0), tile(0, 1)) -> GreenLaser
impulse(tile(0, 1), tile(1, 0)) -> MirrorH
impulse(tile(1, 0), tile(1, 1)) -> SensorA
```

This creates a chain: `GreenLaser → MirrorH → SensorA → FrameBuffer`.

## Invoking a Mesh

`invoke` executes the mesh and optionally pipes the result through a network channel and API:

```hololang
invoke(HoloMesh)
invoke(HoloMesh) -> HoloGRPC -> HoloAPI
```

## Canvas Declaration

A `canvas` is a named surface that can wrap a mesh:

```hololang
canvas HoloCanvas(HoloMesh) {
    self.visible = true
}
```

> **Note:** To set properties on the canvas object at runtime, assign directly to `self` inside the canvas block (e.g., `self.visible = true`).

## CLI Canvas Rendering

```bash
hololang canvas examples/hello_holo.hl        # colour terminal output
hololang canvas examples/hello_holo.hl --no-color
```
