# Virtual Machine

HoloLang programs run on a stack-based kernel VM with configurable cycle limits, worker pools, and a block-controller generative pipeline.

## Kernel

```hololang
kernel MainKernel {
    stack_limit: 512       // maximum stack depth
    max_cycles:  100000    // halt after N instruction cycles
    replicas:    4         // number of replicated kernel instances
}
```

## Worker Pool

A `pool` groups kernels into a parallel worker fleet:

```hololang
pool HoloPool {
    kernels: MainKernel
    workers: 8
    mode:    ProcessMode.REALTIME   // enum value
}
```

Pool `mode` options:

| Value | Behaviour |
|---|---|
| `REALTIME` | Low-latency, immediate dispatch |
| `DEFERRED` | Queue work, flush at end of frame |
| `BATCH` | Bulk-process accumulated work |

## Runtime

```hololang
runtime HoloVM {
    pool:     HoloPool
    surfaces: [FrameBuffer, BeamWeights, SensorData]
}
```

`surfaces` lists the tensors the VM can read and write.

## Execution Order

1. Top-level statements evaluated left-to-right, top-to-bottom
2. `invoke(Mesh)` triggers tile evaluation in row-major order
3. `impulse` cycles fire in declaration order within each invoke
4. `emit` statements dispatch network messages after mesh evaluation
