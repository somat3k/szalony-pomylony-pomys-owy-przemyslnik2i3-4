# Tensors

HoloLang has a built-in tensor subsystem supporting N-dimensional arrays with optional bounds-checking (SafeTensor), computation graphs, and pooled parallel runtimes.

## Basic Tensor

```hololang
tensor FrameBuffer[1080][1920] {
    dtype: "float32"    // "float32" | "float64" | "int32" | "int64" | "uint8"
}
```

## SafeTensor

Add `min_val`, `max_val`, and `clamp: true` to enable automatic bounds-checking:

```hololang
tensor FrameBuffer[1080][1920] {
    dtype:   "float32"
    min_val: 0.0
    max_val: 1.0
    clamp:   true
    pool:    "replicated"
}
```

When `clamp: true`, any write that exceeds `[min_val, max_val]` is silently clamped. When `clamp: false`, an out-of-bounds write raises a `ValueError`.

## Supported dtypes

| `dtype` | NumPy equivalent | Notes |
|---|---|---|
| `"float32"` | `np.float32` | Default for beam data |
| `"float64"` | `np.float64` | High-precision computations |
| `"int32"` | `np.int32` | Indices, counters |
| `"int64"` | `np.int64` | Large index spaces |
| `"uint8"` | `np.uint8` | Raw image buffers |

## Computation Graph

Tensors can be wired into a directed computation graph:

```hololang
tensor InputA[4][4] { dtype: "float32" }
tensor InputB[4][4] { dtype: "float32" }
tensor Output[4][4] { dtype: "float32" }

graph MatMulPipeline {
    matmul(InputA, InputB) -> relu -> normalize -> Output
}
```

Supported graph operations: `matmul`, `relu`, `normalize`, `transpose`, `add`, `mul`.

## Pool Runtime

The `pool` property on a tensor declares it as a replicated surface distributed across a worker pool:

```hololang
tensor FrameBuffer[1080][1920] {
    dtype: "float32"
    pool:  "replicated"
}

pool ScanPool {
    kernels: ScanKernel
    workers: 4
}
```
